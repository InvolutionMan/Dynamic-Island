# Fantastic Island v0.1.0

First public release.

Fantastic Island is an open, extensible app for the macOS notch area, built around a shell architecture with modular capabilities. This release is published as `source-only` and is meant to establish the current container, built-in modules, and extension direction as the starting point for future work.

## Highlights

- A modular island shell that can act as the container for custom capabilities
- `Codex` module for local Codex sessions, quota, approvals, tool activity, and notifications
- `ClashX Meta` module for Mihomo / ClashX Meta runtime integration, with room to extend traffic, proxy groups, rules, connections, and logs
- `Player` module for current playback state, artwork, progress, and basic media controls
- `Wind Drive`, a playful fan component with configurable logo, sound, and presentation behavior
- A settings surface and design token debugging entry for app behavior, module toggles, and visual tuning

## Repository Scope

Included in this first release:

- Full application source code
- Xcode project
- Current built-in module implementations
- License and third-party notices

Not included in this first release:

- DMG packaging artifacts
- Code signing or notarization setup
- Personal development team identifiers
- Prebundled ClashX Meta runtime, dashboard, or geodata assets

If you want the full managed ClashX Meta workflow, you need to provide compliant runtime assets yourself.

## Build

```bash
xcodebuild -project 'FantasticIsland/FantasticIsland.xcodeproj' -scheme 'FantasticIsland' -configuration Debug CODE_SIGNING_ALLOWED=NO build
```

Lightweight logic tests:

```bash
swift test
```

## Notes

The notch shell interaction layer in Fantastic Island inherits and adapts parts of `open-vibe-island`. This release should be treated as a public starting point: stabilize the shell, module structure, and current built-in capabilities first, then keep pushing toward a more customizable and extensible island.
